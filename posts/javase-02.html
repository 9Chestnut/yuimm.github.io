JVM

  • 虚拟机栈

    执行殷引擎每调用一个函数，就会为这个函数创建一个栈帧，并加入虚拟机栈。即每个函数从调用到执行结束，其实就是对应栈帧的入栈和出栈。

  • 堆

    被所有的线程所共享的一块区域，在虚拟机启动时创建，所有的对象实例及数组都在堆上分配。

    使用new关键字，表示在堆中开辟一块新的存储空间。

  • 方法区

    也称静态区，存放所有的class字节码和static变量，方法区中包含的都是在程序中永远唯一的元素。

  • 程序计数器

  • 本地方法区

    与虚拟机栈类似，为Native方法服务。

引用数据类型

​ —–类、接口、数组、枚举、标注

  • 基本数据类型是值传递

  • 引用类型传递是地址的传递

构造器

  • 方法名和类名相同的特殊方法就是构造器

  • 一个类至少存在一个构造器

  • 禁止定义返回类型，也不需要使用return语句

  • 构造器其实有返回值，返回的是当前创建对象的地址

  • 作用：

    创建对象

    完成对象的初始化操作

默认构造器

  • 无参数、方法体
  • 如果类没有public修饰，那么默认构造器也没有

自定义构造器

  • 定义了构造器，编译器不会再创建默认构造器
  • 创建对象其实是再调用构造器

匿名对象

  • 只能使用一次

static关键字

  • 通过static修饰的成员属于类本身，不属于对象
  • 可以修饰字段，方法，内部类
  • 作用：确定修饰的内容是属于类还是属于对象
  • 随着类加载而加载
  • 优先于对象存在
  • static修饰的成员被该类型的所有对象共享
  • 可以使用当前类的类名访问static成员
  • 静态方法不需要通过创建对象来调用
  • 静态方法不能访问普通成员，只能访问静态成员
  • 不能在方法中定义静态成员，方法中都属于局部变量，运行时在栈中分配空间，静态成员是在方法区中存储

局部变量和全局变量

  • 局部变量，必须进行初始化才能使用。

封装、继承、多态

封装

  • protexted: 只能在同一个包中使用，但是继承状态下不同包也能用
  • 默认权限继承情况下也只能在同一个包中才能使用

继承

  • 继承中的方法覆盖

  • 方法重载和方法重写的区别

  • 父类中如果自定义了构造器，子类继承时，必须调用此构造器

  • 子类初始化过程

    创建子类对象之前，会先创造父类对象

    调用子类的构造器之前，在子类构造器中会先调用父类的构造器

  • 父类中的静态方法和子类中的静态方法是一摸一样的 ？？称为方法的隐藏

多态

在方法调用时，通过父类进行调用，真正执行时，调用的是子类的方法，这种特征称之为多态。

一个对象有多种形态。

  • 把子类对象赋值给父类的类型

    Animal dog = new Dog();

  • 作用：

    当把不同的子类对象当作父类类型来看待，可以屏蔽不同子类对象之间的实现差异。从而写出通用的代码达到通用编程，以适应需求的不断变化。

  • 注意问题

    如果父类中没有要调用的方法，但是子类中有，执行时，如果是多态就会报编译的错误，如果要使用，就强转为子类类型。

    如果父类和子类中都有，并且此方法是静态方法，多态情况下，调用的是父类中的静态方法。

instanceof

判断一个对象是否为指定的类 

if(dog instanceof Animal){}

UML（标准建模语言）

用来表示软件开发过程中类与类之间的关系

代码块与final关键字

局部代码块

  • 直接定义在方法内部。

初始化代码块

  • 调用构造器，会把初始化代码块放在构造器当中。
  • 每创建一个对象（构造器执行一次），就会执行一次

静态代码块

  • 在把字节码加载到jvm的时候执行，比main方法还早。
  • 只执行一次

类什么时候被加载

当第一次使用该类的时候，就会把该类对象的字节码加载到jvm中

继承状态下，先加载父类再加载子类

final关键字

  • 不可修改的，保证数据的安全
  • 可修饰字段、方法（子类不能覆盖）、类（类不能被继承）
  • 修饰字段时，没有初始值，必须手动设置初始值
  • final修饰的字段可在局部代码块使用
  • 修饰引用类型，表示地址不能再修改

包装类

对基本数据类型进行包装成一个对象。

把基本数据类型变得更强大，以面向对象的思想去使用这些类型。

装箱

把基本数据类型 >> 包装类

两种操作：

Integer num1 = new Integer(10);

Integer num1 = Integer.valueOf(10);

可以把一个字符串转成Integer类型，字符串必须是数字。

拆箱

把包装类 >> 基本数据类型

自动装箱

可以直接把一个基本数据类型的值赋值给包装类对象。

自动拆箱

可以直接把一个包装类对象赋值给基本数据类型。

String与其他类型转换

通过toString()方法把包装类对象转为字符串

包装类与基本数据类型的区别

  • 默认值不同
  • 包装类中提供了很多方法直接使用
  • 集合框架中不能存放基本数据类型，只能存对象
  • 类中成员变量一般使用包装类，方法中一般使用基本数据类型

包装类valueOf缓存设计

缓存的数据范围只有 -128~127

抽象类与接口

抽象方法

  • 在方法前面加一个关键字abstract
  • 没有方法体
  • 必须定义在抽象类中
  • 子类必须覆盖父类中的所有抽象方法

抽象类

  • 抽象类要有子类才行
  • 不能创建对象
  • 抽象类中可以有抽象方法，也可以有非抽象方法
  • 把子类变成抽象类就可以不覆盖方法

接口

  • 没有构造器，不能创建对象
  • 接口中定义变量，都是全局的静态常量
  • 接口可以多继承
  • 面向接口编程，其实就是多态

接口与抽象类区别

  • 抽象类有构造器，接口中没有构造器
  • 抽象类可以包含普通方法和抽象方法，接口中只有抽象方法
  • 接口中成员变量权限为：public static final，抽象类中是默认的权限

内部类与枚举

内部类

定义在类中的一个类

  • 增强封装
  • 增加代码的维护性
  • 可以直接访问外部类当中的成员

内部类的划分

  • 实例内部类88

    直接定义在类中，类前面没有任何修饰符

  • 静态内部类

    再内部类前面加一个static

  • 局部内部类

    定义再方法中的内部类

    只能在方法中使用

    不能包含静态变量

    可以使用局部变量，局部变量本质是final

  • 匿名内部类

    属于局部内部类的一种特殊情况

    没有构造器 new父类的构造器或者接口

枚举

表示有个事物固定的状态

是一个特殊的类，多个常量对象的集合

[] enum 枚举的名称 {常量1，常量2…………}

对构造器进行了私有化，不能再创建对象。

集合

  • 长度可以改变
  • 能存储任意的对象
  • 集合只能存引用数据类型，如果放基本数据类型，会自动装箱。

Collection

  • List（有序）

    • ArrayList 数组
    • LinkedList 链表
    • Vector 数组,都会加锁

  • Set（无序）（保证元素唯一）

    • HashSet Hash算法

    • linkedHashSet

    • TreeSet 二叉树。

      会对元素排序，自定义对象不能直接添加，必须实现Comparable接口,并且覆盖其中的compareTo方法，如果返回值是一个正数，都可以添加，添加到右树，而且是正序，返回0只添加一个元素，返回一个负数，也会全添加进去，添加到左树，但是是倒序。

ArrayList

  • 可以添加重复的元素
  • 覆盖了toString方法
  • 对基本数据类型自动装箱
  • toArray会把数组中的元素向上转型为Object类型

MAP

  • 无序
  • map元素的遍历
    1. 通过Iterator或者foreach
    2. 通过entry

迭代器 Iterator

collection的每一个实现类中都定义了自己的迭代方法

list

  • list可以根据角标操作元素

并发修改异常

泛型

  • 广泛通用的类型

  • 代码中类型不确定，谁调用谁就来指定类型，没有指明就是Object类型

  • 在类中定义的泛型，只能用在普通方法上，不能用在静态方法上面

  • 方法想要单独使用泛型，必须得有参数才有意义

  • 通配符

    不知道使用什么类型来接收时，可以使用？表示未知，只能用来做接收

    使用

  • 泛型的上下限

    extends：必须是指定类或者指定类型的子类

    super：必须是指定类或者指定类型的父类

  • 泛型擦除

  • 数组转集合之后，仍不能添加、删除元素，转过之后，可以以面向对象的方式操作数组，会把整个数组当作一个对象。
