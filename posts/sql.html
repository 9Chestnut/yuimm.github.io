第一章 检索记录

1、在 where 子句中引用别名列

你已经为检索结果集创建了有意义的列名，并且想利用 WHERE 子句过滤掉部分行数据。但是，如果你尝试在 WHERE 子句中引用别名列，查询无法顺利执行。

SELECT  template_name as name ,activity_no as no
from pc_activity
where name = '换购'

解决方案
把查询包装为一个内嵌视图，这样就可以引用别名列了。

select * from (
SELECT  template_name as name ,activity_no as no
from pc_activity
) as x
where name = '换购'

#在本例中，内嵌视图的别名为 X。并非所有数据库都需要给内嵌视图取别名， 但对于某些数据库而言，确实必须如此。不过，所有的数据库都支持这一 点。

将含有别名列的查询放入内嵌视图，就可以在外层查询中引用别名列

WHERE 子句会比 SELECT 子句先执行，就最初那个失败的查询例子而言，当 WHERE 子句 被执行时，name和 no 尚不存在。直到 WHERE 子句执行完毕，那些别名列才会生效。然而，FROM 子句会先于 WHERE 子句执行。如果把最初的那个查询放入一个 FROM 子句，其查询结果会在最外层的 WHERE 子句开始之前产生，这样一来，最外层的 WHERE 子句就能 “看见”别名列了。当表里的某些列没有被恰当命名的时候，这个技巧尤其有用。

2、串联多列的值（把两列的值合并为一列展示）

# MySql使用concat函数
select concat(activity_name,' activityNo is ',activity_no) as msg
from pc_activity
where activity_type = 1

# DB2、Oracle 和 PostgreSQL 这些数据库把双竖线作为串联运算符
select activity_name || ' activityNo is ' || activity_no as msg
from pc_activity
where activity_type = 1

#SQL Server 该数据库使用“+”作为串联运算符。
select activity_name + ' activityNo is ' + activity_no as msg
from pc_activity
where activity_type = 1

3、在SELECT语句里使用条件逻辑

select activity_no,activity_type,
    case when activity_type = 1 then '单品促销'
         when activity_type = 2 then '满减促销'
         when activity_type = 8 then '换购促销'
         else '秒杀'
  end as status
from pc_activity

4、限定返回行数

你想限定查询结果的行数。你不关心排序，任意 n 行都可以

解决方案
使用数据库的内置功能来控制返回的行数。

# MySQL 和 PostgreSQL使用 LIMIT 子句。
select * 
from pc_activity
limit 5        

# DB2 使用 FETCH FIRST 子句。
select*
from pc_activity
fetch first 5 rows only

# Oracle
select *    
from pc_activity
where rownum <= 5

当你使用 ROWNUM<=5 限定只返回最初的  5 行数据时，会发生如下的事情:
Oracle 取得第一行数据，并把它的行号定为 1
已经超过第 5 行了吗？如果没有，Oracle 会返回当前行，因为当前的行号满足小于或等 于  5 这一条件
Oracle 取得下一行数据，并且将行号加 1（得到 2，然后得到 3,以此 类推

如果指定 ROWNUM=5，希望只返回第 5 行 会失败
如果你不先获取第 1 行到第 4 行，第 5 行从何而来？
ROWNUM=1 作为条件会成功！

5、随机返回若干行记录

解决方案
使用数据库的内置函数来随机生成查询结果

# DB2 把内置函数 RAND 和 ORDER BY、FETCH 结合使用
select activity_no,activity_type,
from pc_activity
order by rand() fetch first 5 rows only

# MySQL 把内置函数 RAND 和 LIMIT、ORDER BY 结合使用。
select activity_no,activity_type,
from pc_activity
order by rand() limit 5

# PostgreSQL 把内置函数 RANDOM 和 LIMIT、ORDER BY 结合使用。
select activity_no,activity_type,
from pc_activity
order by random() limit 5

# Oracle 在内置包 DBMS_RANDOM 里可以找到 VALUE 函数，把该内置函数和 ORDER BY、内置函数 ROWNUM 结合使用。
select *
from (selectactivity_no,activity_type,
    from pc_activity 
    order by dbms_random.value() )
where rownum <= 5

6、查找Null值

select *
from pc_activity
where activity_description is null

select *
from pc_activity
where activity_description is not null

7、把Null值转换为实际值

有一些行包含 Null 值，但是你想在返回结果里将其替换为非 Null 值。

解决方案
使用 COALESCE 函数将 Null 值替代为实际值。

处理 Null 值时，最好利用数据库的内置功能。在许多情况下，你会发现有不止一个函数能 解决本实例中的问题COALESCE 函数只是恰好适用于所有的数据库。除此之外，CASE 也适 用于所有数据库。

select coalesce(activity_description,'000')
from pc_activity

select 
    case when activity_description is not null 
                then activity_description
         else '新加的'
         end
from pc_activity

8、查找匹配项

你想返回匹配某个特定字符串或模式的行。考虑下面的查询及其结果集

select activity_name,activity_type,create_user_name
from pc_activity
where activity_type in(2,8)
    and (activity_name like '%测试%' or create_user_name like '%1')

用于 LIKE 模式匹配操作时，运算符 % 可以匹配任意长度的连续字符。大多数 SQL 实现 也提供了下划线（_）运算符，用于匹配单个字符

第二章 查询结果排序

1、以指定顺序返回查询结果

解决方案
使用 ORDER BY 子句

select activity_name,activity_type
from pc_activity
where activity_type in(1,2,8)
order by activity_type asc

默认情况下，ORDER BY 会做升序排列，也可以通过指定 DESC 执行降序排列。

你也可以不指定用于排序的列名，而指定一个数值来指代该列。数值从 1 开始，从左向右 匹配 SELECT 列表里的列，如下所示。

select activity_name,activity_type
from pc_activity
where activity_type in(1,2,8)
order by 2 desc

上述 ORDER BY 子句里的数字 2 对应着 SELECT 列表的第 2 列，即 activity_type

2、多字段排序

先按照其中一列数据做升序排列，然后再按照 另一列数据做 降序排列

解决方案
在 ORDER BY 子句中列出不同的排序列，以逗号分隔。

select activity_name,activity_type
from pc_activity
where activity_type in(1,2,8)
order by activity_name ,activity_type desc

ORDER BY 的执行顺序是从左到右的

3、依据子串排序

按照一个字符串的特定部分排列查询结果，例如，你希望从 EMP 表检索员工的名字和 职位，并且按照职位字段的最后两个字符对检索结果进行排序。

解决方案

# DB2、MySQL、Oracle 和 PostgreSQL 在 ORDER BY 子句里使用 SUBSTR 函数。
select activity_name,activity_type,create_user_name
from pc_activity
where activity_type in(1,2,8)
order by substr(activity_name,length(activity_name)-2)

# SQL Server 在 ORDER BY 子句里使用  SUBSTRING 函数。
select activity_name,activity_type,create_user_name
from pc_activity
where activity_type in(1,2,8)
order by substring(activity_name,length(activity_name)-2,2)

利用数据库中的子串函数，你可以很方便地按照一个字符串的任意部分来排序。要想按照 一个字符串的最后两个字符排序，需要先找到该字符串的结尾处（即字符串的长度），然后减去 2。这样，起始位置就是该字符串的倒数第 2 个字符。然后，你就可以截取从指定 起始位置开始直到字符串结束的所有字符。SQL Server 的 SUBSTRING 函数略有不同，它要 求提供第 3 个参数来指定需要截取几个字符。对于本实例而言，第 3 个参数既可以是 2， 也可以是任何大于 2 的数字。

⭐4、对含有字母和数字的列排序⭐

有混合了字母和数字的数据，希望按照字母部分或者数字部分来排序。

5、排序时对Null值的处理

对查询结果进行排序，但该字段可能为 Null。是否应该将 Null 值排到后面。

解决方案

使用 CASE 表达式来动态调整排序项。

# DB2、MySQL、PostgreSQL 和 SQL Server 
select activity_name,activity_type,activity_description
from (
    select activity_name,activity_type,activity_description,
        case
            when activity_description is null then 0
                else 1
                end as is_null
    from pc_activity
    ) as x
order by is_null desc,activity_description

使用 CASE 表达式标记 Null 值。该标记有两种可能的取值：一种代表 Null 值，另一种代 表非 Null 值。一旦你做好了标记，只要简单地把它放进 ORDER BY 子句就行了。这样一来， 你就能在不影响非 Null 值的情况下，方便地调整 Null 值的位置了。

6、依据条件逻辑动态调整排序项

希望按照某个条件逻辑来排序。例如，如果 JOB 等于 SALESMAN，就要按照 COMM 来排序； 否则，按照 SAL 排序

解决方案
在 ORDER BY 子句里使用 CASE 表达式。

select activity_name,activity_type,activity_no,id
from pc_activity
order by 
    case when activity_type = 2 then 'activity_no'
                else 'id' 
                end 
SELECT
    activity_name,
    activity_type,
    activity_no,
    ID,
CASE

        WHEN activity_type = 2 THEN
        'activity_no' ELSE'id' 
    END 
    FROM
        pc_activity 
    ORDER BY
        4
